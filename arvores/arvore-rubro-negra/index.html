Estruturas de dados · Árvore balanceada

Árvore
Rubro-Negra

Uma árvore binária de busca auto-balanceada que usa coloração de nós para garantir complexidade O(log n) em todas as operações. Criada por Rudolf Bayer em 1972 e nomeada por Guibas & Sedgewick em 1978.

Inserção O(log n) Remoção O(log n) Busca O(log n) Max 3 rotações por remoção

Fundamentos

Propriedades da árvore

Toda árvore rubro-negra deve satisfazer simultaneamente todas estas propriedades. Qualquer violação exige correção por rotação ou troca de cor.

1

Coloração dos nós

Todo nó da árvore possui uma cor: vermelho ou preto. Não existe terceira opção.

2

Raiz sempre preta

O nó raiz é invariavelmente preto. Após qualquer operação, a raiz é recolorida para preto se necessário.

3

Folhas nulas são pretas

Todo nó folha NULL (sentinela) é considerado preto. Isso simplifica as regras de balanceamento.

4

Filhos de vermelho são pretos

Se um nó é vermelho, obrigatoriamente ambos seus filhos são pretos. Não existem nós vermelhos consecutivos.

5

Altura negra uniforme

Para cada nó, todos os caminhos até as folhas descendentes contêm exatamente o mesmo número de nós pretos.

★

LLRB — regra extra

Na variante Left-Leaning (Sedgewick, 2008), nós vermelhos são sempre filhos à esquerda do pai.

Comparativo

AVL vs Rubro-Negra

Ambas têm a mesma complexidade assintótica, mas diferem significativamente na prática. A escolha depende do perfil de uso da aplicação.

Regra prática: Se sua aplicação realiza buscas com muito mais frequência que inserções/remoções → use AVL. Se inserções e remoções são frequentes → use Rubro-Negra. Ela é preferida em bibliotecas de linguagens de programação por ser de uso mais geral.

Balanceamento

Rotações e troca de cores

A árvore rubro-negra utiliza apenas duas funções de rotação (complexidade O(1) cada) e a troca de cores para corrigir violações após inserção ou remoção.

🔄 Rotação à esquerda

Recebe nó A com B como filho direito. Move B para o lugar de A; A se torna filho esquerdo de B. B herda a cor de A e A fica vermelho.

NO* rotacionaEsquerda(NO* A){ NO* B = A->dir; A->dir = B->esq; B->esq = A; B->cor = A->cor; A->cor = RED; return B; }

🔄 Rotação à direita

Recebe nó A com B como filho esquerdo. Move B para o lugar de A; A se torna filho direito de B. B herda a cor de A e A fica vermelho.

NO* rotacionaDireita(NO* A){ NO* B = A->esq; A->esq = B->dir; B->dir = A; B->cor = A->cor; A->cor = RED; return B; }
Correções de violações na inserção
CASO 1
Filho direito vermelho
Filho esquerdo preto → rotação à esquerda
CASO 2
Filho + neto esquerdo vermelhos
Dois vermelhos à esquerda → rotação à direita
CASO 3
Ambos os filhos vermelhos
Dois filhos vermelhos → troca de cores
