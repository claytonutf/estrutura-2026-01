// fundamentos

Por que balancear?

Algoritmos de inserção e remoção em árvores binárias comuns não garantem balanceamento. No pior caso, a árvore vira uma lista e as operações passam de O(log N) para O(N).

⚖️

Fator de Balanceamento

Diferença entre a altura da sub-árvore esquerda e direita de um nó. Em uma AVL, esse valor deve ser sempre –1, 0 ou +1.

🔁

Rebalanceamento Local

Apenas a parte afetada pela inserção ou remoção é rebalanceada, usando rotações simples ou duplas.

📐

Árvore Quase Completa

As rotações buscam manter a árvore próxima de uma forma completa, garantindo que qualquer operação tenha custo O(log N).

🧮

Altura da Sub-árvore

Se a sub-árvore for nula (NULL), sua altura é considerada –1. Caso contrário, é o valor armazenado no campo alt do nó.

// fórmula
FB = AE – AD
AE = altura sub-árvore esquerda  ·  AD = altura sub-árvore direita
// valores válidos
FB = –1 FB = 0 FB = +1 FB = ±2 → rotação!
// operações

Os 4 tipos de rotação

Rotações são aplicadas no ancestral mais próximo do nó inserido/removido cujo fator de balanceamento se torna ±2. Clique em cada rotação para ver os detalhes.

LL

Rotação Simples Direita

Nó inserido na sub-árvore esquerda do filho esquerdo. Dois passos à esquerda.

RR

Rotação Simples Esquerda

Nó inserido na sub-árvore direita do filho direito. Dois passos à direita.

LR

Rotação Dupla Direita

Nó inserido na sub-árvore direita do filho esquerdo. RR em B + LL em A.

RL

Rotação Dupla Esquerda

Nó inserido na sub-árvore esquerda do filho direito. LL em B + RR em A.

// tabela de decisão

Quando usar cada rotação

Sinais iguais → rotação simples. Sinais opostos → rotação dupla.

FB do nó A FB do filho B Posição Rotação
+2 +1 B = filho esq. de A · C = filho esq. de B LL
–2 –1 B = filho dir. de A · C = filho dir. de B RR
+2 –1 B = filho esq. de A · C = filho dir. de B LR
–2 +1 B = filho dir. de A · C = filho esq. de B RL

Complexidade de tempo

Graças ao balanceamento automático, todas as operações principais são garantidas em O(log N), independente da ordem de inserção.

Busca
O(log N)
melhor, médio e pior caso
Inserção
O(log N)
inclui rotação se necessário
Remoção
O(log N)
inclui rotação se necessário
BST não balanceada
O(N)
pior caso: entrada ordenada
