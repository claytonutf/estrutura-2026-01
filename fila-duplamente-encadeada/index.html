01 — Fundamentos

O que é uma Lista
Duplamente Encadeada?

Uma lista duplamente encadeada é uma estrutura de dados onde cada elemento (nó) armazena dados e dois ponteiros: um para o nó seguinte (prox) e outro para o nó anterior (ant). Isso permite navegação nos dois sentidos.

Neste exemplo, a estrutura é usada para implementar uma fila (queue) — onde inserções ocorrem no final (enqueue) e remoções no início (dequeue).

🔗

Navegação Bidirecional

Ao contrário da lista simples, é possível percorrer do início ao final e do final ao início com o ponteiro ant.

⚡

Inserção e Remoção O(1)

Com os ponteiros inicio e final na estrutura da fila, inserir no final e remover no início são operações em tempo constante.

🧠

Alocação Dinâmica

Cada nó é criado com new em tempo de execução — a lista cresce e encolhe conforme necessário, sem tamanho fixo.

02 — Definição das Structs

Como os Dados São
Organizados em Memória

A implementação usa três estruturas encadeadas: aluno (os dados), elemento (o nó) e fila (o controle).

struct elemento ← nó

  • aluno dados
  • elemento *prox →
  • elemento *ant ←

O nó da lista. Aponta para os vizinhos.

struct fila

  • elemento *inicio
  • elemento *final

Controla as extremidades da fila.

💡 typedef fila* Fila — O código usa typedef para criar o alias Fila, tornando as funções mais legíveis ao trabalhar com ponteiros para a struct.

04 — Funções Implementadas

Todas as Operações
da Fila

Função Operação Complexidade Descrição
cria_fila() Criação O(1) Aloca a struct fila e inicializa inicio e final como nullptr.
insere_fila(fi, al) Enqueue O(1) Insere um aluno no final da fila. Ajusta ponteiros prox e ant do novo nó e do antigo final.
remove_fila(fi) Dequeue O(1) Remove o primeiro elemento. Atualiza inicio e zera o ponteiro ant do novo início.
consulta_inicio(fi, al) Peek Front O(1) Copia os dados do primeiro elemento sem removê-lo. Retorna 0 se vazia.
consulta_final(fi, al) Peek Back O(1) Copia os dados do último elemento sem removê-lo.
tamanho_fila(fi) Size O(n) Percorre toda a lista contando nós. Poderia ser O(1) com um contador interno.
imprime_inicio_final(fi) Print → O(n) Percorre do início ao final usando ponteiros prox.
imprime_final_inicio(fi) Print ← O(n) Percorre do final ao início usando ponteiros ant — exclusivo da lista dupla!
libera_fila(fi) Free O(n) Remove todos os nós com delete e libera a struct fila. Evita memory leak.

06 — Implementação Completa

Código Comentado
em C++

// ─── DADOS DO ALUNO ───────────────────────────────
struct aluno {
    int   matricula;     // identificador
    char  nome[30];      // nome
    float n1, n2, n3;  // notas
};

// ─── NÓ DA LISTA ──────────────────────────────────
// Estrutura DUPLAMENTE encadeada:
//   ant ← [dados] → prox
struct elemento {
    aluno     dados;  // payload
    elemento *prox;  // → próximo nó
    elemento *ant;   // ← nó anterior
};

// ─── CONTROLE DA FILA ─────────────────────────────
// Guarda ponteiros para as extremidades
struct fila {
    elemento *inicio;  // primeiro (dequeue aqui)
    elemento *final;   // último  (enqueue aqui)
};

typedef fila* Fila;  // alias para ponteiro

// ─── CRIAÇÃO ──────────────────────────────────────
Fila cria_fila() {
    Fila fi = new fila;
    if (fi != nullptr) {
        fi->inicio = nullptr;
        fi->final  = nullptr;
    }
    return fi;
}
// ─── ENQUEUE — Inserção no Final ──────────────────
// Complexidade: O(1)
int insere_fila(Fila fi, aluno al) {

    if (fi == nullptr) return 0;

    // 1. Aloca novo nó
    elemento *no = new elemento;
    if (no == nullptr) return 0;

    no->dados = al;
    no->prox  = nullptr; // será o novo final

    // ── CASO 1: fila vazia ──────────────────────────
    if (fi->inicio == nullptr) {
        no->ant    = nullptr;
        fi->inicio = no;
        fi->final  = no;
    }

    // ── CASO 2: fila com elementos ──────────────────
    else {
        no->ant          = fi->final; // ← aponta pro antigo final
        fi->final->prox = no;        // antigo final → novo nó
        fi->final        = no;        // atualiza ponteiro final
    }

    return 1;
}
// ─── DEQUEUE — Remoção do Início ──────────────────
// Complexidade: O(1)
int remove_fila(Fila fi) {

    if (fi == nullptr || fi->inicio == nullptr)
        return 0;

    elemento *no = fi->inicio; // guarda o nó a remover

    // ── CASO 1: apenas 1 elemento ───────────────────
    if (fi->inicio == fi->final) {
        fi->inicio = nullptr;
        fi->final  = nullptr;
    }

    // ── CASO 2: vários elementos ────────────────────
    else {
        fi->inicio       = fi->inicio->prox;
        // novo início não tem anterior
        fi->inicio->ant = nullptr;
    }

    delete no; // libera memória do nó removido
    return 1;
}
// ─── LIBERA FILA — Evita Memory Leak ──────────────
// Complexidade: O(n)
void libera_fila(Fila fi) {

    if (fi == nullptr) return;

    elemento *no;

    // percorre do início ao final deletando cada nó
    while (fi->inicio != nullptr) {
        no            = fi->inicio;
        fi->inicio    = fi->inicio->prox;
        delete no;
    }

    delete fi; // libera a struct fila em si
}

// ─── IMPRESSÃO REVERSA ────────────────────────────
// Só possível graças ao ponteiro ant!
void imprime_final_inicio(Fila fi) {

    elemento *no = fi->final; // começa do FINAL

    while (no != nullptr) {
        cout << no->dados.nome << endl;
        no = no->ant; // ← navega para trás
    }
}
int main() {

    // dados de exemplo
    aluno a[4] = {
        {1, "Andre",   9.5, 7.8, 8.5},
        {2, "Ricardo", 7.5, 8.7, 6.8},
        {3, "Bianca",  9.7, 6.7, 8.4},
        {4, "Ana",     5.7, 6.1, 7.4}
    };

    Fila fi = cria_fila();

    // enqueue: insere 4 alunos no final
    for (int i = 0; i < 4; i++)
        insere_fila(fi, a[i]);

    imprime_inicio_final(fi); // →
    imprime_final_inicio(fi);  // ← reversa!

    // peek: consulta sem remover
    aluno aux;
    consulta_inicio(fi, &aux);
    consulta_final(fi, &aux);

    // dequeue: remove 4 vezes
    for (int i = 0; i < 4; i++)
        remove_fila(fi);

    libera_fila(fi); // sempre libere a memória!
    return 0;
}
07 — Comparação

Lista Simples vs Dupla
vs Vetor

Vetor / Array

  • Tamanho fixo (ou realloc caro)
  • Acesso O(1) por índice
  • Inserção/remoção no meio: O(n)
  • Sem ponteiros adicionais
  • Cache-friendly

Lista Simplesmente Encadeada

  • Tamanho dinâmico
  • Inserção/remoção no início: O(1)
  • Navegação apenas →
  • Sem acesso por índice
  • 1 ponteiro por nó

Lista Duplamente Encadeada

  • Tamanho dinâmico
  • Inserção/remoção nas extremidades: O(1)
  • Navegação → e ←
  • Impressão reversa fácil
  • Ideal para Fila e Deque
  • 2 ponteiros por nó (mais memória)
🎓 Quando usar lista duplamente encadeada? Sempre que precisar de inserções e remoções eficientes em ambas as extremidades, navegação reversa, ou quando o tamanho da coleção é imprevisível em tempo de compilação.
