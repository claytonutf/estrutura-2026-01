01 — Conceito

O que é uma Pilha?

Uma pilha é uma estrutura de dados linear que segue o princípio LIFO — Last In, First Out (Último a Entrar, Primeiro a Sair). Pense em uma pilha de pratos: você só pode adicionar ou remover pelo topo.

📚

LIFO

Last In, First Out — o último elemento inserido é sempre o primeiro a ser removido.

🔝

Topo (Top)

Toda inserção e remoção acontece no topo. A base permanece intocada até que todos os outros sejam removidos.

🔗

Encadeada

Nesta implementação, cada nó armazena um ponteiro para o próximo, criando uma lista encadeada.

⚡

O(1) constante

As operações push, pop e peek são executadas em tempo constante — independente do tamanho da pilha.

03 — Operações

As quatro operações fundamentais

A pilha expõe uma interface simples e bem definida. Todas as operações têm complexidade O(1).

PUSH

Inserção no Topo

Cria um novo nó, copia os dados, aponta prox para o topo atual e o novo nó vira o novo topo.

Complexidade: O(1)
POP

Remoção do Topo

Guarda o ponteiro do topo, avança o topo para prox e libera a memória do nó removido.

Complexidade: O(1)
PEEK

Consulta do Topo

Retorna uma cópia dos dados do topo sem removê-lo. A pilha permanece intacta após a operação.

Complexidade: O(1)
SIZE

Tamanho

Percorre todos os nós contando-os. Nesta implementação encadeada é O(n). Com contador interno seria O(1).

Complexidade: O(n)

05 — Código-fonte C++

Implementação completa

Pilha encadeada em C++ com as funções cria_pilha, insere_pilha, remove_pilha, consulta_topo_pilha e libera_pilha.

// =====================
// Estrutura do aluno
// =====================
struct aluno {
    int   matricula;   // identificador
    char  nome[30];    // nome do aluno
    float n1, n2, n3; // notas
};

// =====================
// Nó da pilha encadeada
// =====================
struct elemento {
    aluno     dados;  // payload
    elemento *prox;   // ponteiro para o próximo
};

// Pilha é um ponteiro para elemento (o topo)
typedef elemento* Pilha;

// =====================
// Cria uma pilha vazia
// =====================
Pilha* cria_pilha() {
    Pilha *pi = new Pilha;
    if (pi != nullptr)
        *pi = nullptr;
    return pi;
}
// =====================
// PUSH — insere no topo
// =====================
int insere_pilha(Pilha *pi, aluno al) {

    if (pi == nullptr) return 0;

    // 1. Aloca novo nó
    elemento *no = new elemento;
    if (no == nullptr) return 0;

    // 2. Copia os dados para o nó
    no->dados = al;

    // 3. Novo nó aponta para o topo atual
    no->prox = *pi;

    // 4. Novo nó VIRA o novo topo
    *pi = no;

    return 1;
}

// Complexidade: O(1) — independe do tamanho
// =====================
// POP — remove do topo
// =====================
int remove_pilha(Pilha *pi) {

    if (pi == nullptr || *pi == nullptr)
        return 0; // pilha vazia ou nula

    // 1. Guarda o endereço do topo atual
    elemento *no = *pi;

    // 2. Topo avança para o próximo nó
    *pi = no->prox;

    // 3. Libera a memória do nó removido
    delete no;

    return 1;
}

// Complexidade: O(1) — independe do tamanho
// =====================
// PEEK — consulta o topo
// (sem remover)
// =====================
int consulta_topo_pilha(Pilha *pi, aluno *al) {

    if (pi == nullptr || *pi == nullptr)
        return 0;

    // Copia os dados do topo para *al
    // A pilha NÃO é modificada
    *al = (*pi)->dados;

    return 1;
}

// Complexidade: O(1)
// Diferença do POP: não chama delete
int main() {

    // Dados de exemplo
    aluno a[4] = {
        {1, "Andre",   9.5, 7.8, 8.5},
        {2, "Ricardo", 7.5, 8.7, 6.8},
        {3, "Bianca",  9.7, 6.7, 8.4},
        {4, "Ana",     5.7, 6.1, 7.4}
    };

    Pilha *pi = cria_pilha();

    // PUSH de todos os alunos (i=0..3)
    // Ana será o topo (inserida por último)
    for (int i = 0; i < 4; i++)
        insere_pilha(pi, a[i]);

    // PEEK — consulta sem remover
    aluno aux;
    if (consulta_topo_pilha(pi, &aux))
        cout << aux.nome << endl; // "Ana"

    // POP de todos os elementos
    for (int i = 0; i < 4; i++)
        remove_pilha(pi); // Ana, Bianca, Ricardo, André

    libera_pilha(pi);
    return 0;
}

06 — Aplicações

Onde a pilha é usada?

A pilha é uma das estruturas mais fundamentais da computação, presente em dezenas de contextos reais.

🖥️

Chamadas de função

O sistema operacional usa uma call stack para rastrear qual função chamou qual e onde retornar.

↩️

Desfazer / Refazer

Editores de texto usam pilhas para implementar Ctrl+Z (undo) e Ctrl+Y (redo) de ações.

🌐

Histórico do navegador

O botão "Voltar" do browser funciona como um pop em uma pilha de URLs visitadas.

🧮

Avaliação de expressões

Compiladores usam pilhas para avaliar expressões matemáticas e converter entre notações.

🔍

DFS em grafos

A busca em profundidade (Depth-First Search) usa pilha para rastrear os vértices a visitar.

⚙️

Parser de sintaxe

Verificação de parênteses e chaves balanceados usa pilha para empilhar aberturas e desempilhar ao fechar.
